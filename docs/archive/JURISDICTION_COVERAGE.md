## Rule Schema Standards ğŸ“‹

### Required Fields (All Rules)
```json
{
  "metadata": {
    "name": "String - Human readable",
    "description": "String - What this rule does",
    "effective_date": "YYYY-MM-DD",
    "citations": ["Array of rule citations"],
    "jurisdiction_type": "federal|state|local",
    "state": "String (if state)",
    "court_level": "district|superior|circuit|supreme|appellate"
  },
  "trigger": {
    "type": "ENUM - Trigger type",
    "required_fields": []
  },
  "deadlines": [],
  "dependencies": [],
  "validation": {},
  "settings": {}
}
```

### Deadline Priority Guidelines
- **FATAL**: Jurisdictional deadlines (answer, notice of appeal)
- **CRITICAL**: Court-ordered/statute-mandated (discovery, pretrial)
- **IMPORTANT**: Procedural with consequences (motions, oppositions)
- **STANDARD**: Best practice (internal deadlines)
- **INFORMATIONAL**: Reminders (calendar holds)

## Contributing New Jurisdictions ğŸ¤

### Step 1: Research
1. Find official court rules (state website or Cornell LII)
2. Identify key triggers (complaint served, trial date, etc.)
3. Extract deadline calculations (days, service extensions)
4. Note any unique requirements (Texas Monday Rule, etc.)

### Step 2: Create Rule Schema
1. Copy template from `seed_comprehensive_rules.py`
2. Fill in metadata (name, citations, effective date)
3. Define required fields for trigger
4. List all deadlines with:
   - Offset days
   - Priority
   - Rule citation
   - Conditions (if applicable)

### Step 3: Test & Validate
1. Create test cases
2. Run dry-run executions
3. Verify calculations
4. Get attorney review

### Step 4: Submit
1. Add to seed script
2. Update this coverage document
3. Commit with descriptive message
4. Tag as `jurisdiction: [state/federal]`
